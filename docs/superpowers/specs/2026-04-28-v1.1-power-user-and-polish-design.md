# Skip Cam Login — v1.1 Power-User & Polish Design

**Date:** 2026-04-28
**Status:** Draft for review
**Targets:** Chrome Extension Manifest V3, no new host permissions, no new API permissions

## Goals

Add seven features to v1.0 that fix invisible-state UX problems and add power-user controls, without expanding the extension's domain reach or permission surface. Concretely:

- D2 — Manual "pause this tab" gesture (currently only logout triggers tab suppression)
- D3 — Configurable click delay (currently always instant)
- F1 — Dark mode popup (currently light-only)
- F2 — Toolbar icon badge reflecting per-tab state (currently no visible state)
- F3 — Quick timed pause (15m/1h/4h) from popup
- F4 — Lifetime "clicks saved" counter in popup footer
- F5 — Recent-activity log in popup (collapsible)

## Non-Goals

- No new sites, no Raven/SAML flow handling (those are a separate v1.2+ track).
- No `contextMenus`, no `alarms`, no new host permissions.
- No per-account profiles, no settings export/import (rejected during brainstorm).
- No "log out everywhere" button (rejected — out of scope).

## Architecture Overview

### Permission delta vs v1.0
**No additions.** All seven features fit within existing `storage` + `notifications` permissions and the Manifest V3 `action` API.

### Storage schema additions

`chrome.storage.sync` (settings, syncs across devices):
- *(unchanged)* `moodle_status`, `panopto_status`, `notification_status`, `logout_scope`
- **NEW** `click_delay_ms`: number ∈ {0, 500, 1000, 3000}, default `0`

`chrome.storage.session` (cleared when browser closes):
- *(unchanged)* `suppress_auto_login`: boolean (set on logout, scope=session)
- **NEW** `pause_until`: number (epoch ms timestamp) — F3 timed pause expiry

`chrome.storage.local` (persists, doesn't sync — picked over sync to avoid quota churn from log writes):
- **NEW** `clicks_saved`: integer counter, monotonic
- **NEW** `activity_log`: array of `LogEntry`, FIFO-capped at 50 most recent

`window.sessionStorage` (per-tab, in content script):
- *(unchanged)* `skipcam_suppress_auto_login`: '1' when set on logout
- **NEW** `skipcam_manual_pause`: '1' when set by D2 popup button

### LogEntry shape (F5)
```js
{
  ts: 1745843400000,           // epoch ms
  site: 'Moodle' | 'Panopto',
  action: 'clicked' | 'suppressed' | 'failed',
  reason?: string              // present for 'suppressed' | 'failed'
}
```

`reason` values for `suppressed`: `'logout-session'`, `'logout-tab'`, `'manual-tab'`, `'timed-pause'`, `'site-disabled'`.
`reason` values for `failed`: `'button-not-found'`, `'click-threw'`.

### Suppression source-of-truth (priority, highest first)

When deciding whether to auto-click on a login page, evaluate in order:

1. Site disabled in settings (`<site>_status === false`) → skip, log `suppressed/site-disabled`
2. F3 timed pause active (`pause_until > Date.now()`) → skip, log `suppressed/timed-pause`
3. F3 timed pause expired (`pause_until ≤ Date.now()` and key still present) → clear key, fall through
4. Logout-session suppression (`storage.session[suppress_auto_login] === true` and scope is 'session') → skip, log `suppressed/logout-session`
5. Logout-tab suppression (`sessionStorage[skipcam_suppress_auto_login] === '1'`) → skip, log `suppressed/logout-tab`
6. Manual D2 pause (`sessionStorage[skipcam_manual_pause] === '1'`) → skip, log `suppressed/manual-tab`
7. Otherwise → click after `click_delay_ms`, log `clicked` and increment `clicks_saved`

This replaces the existing single `isSuppressed()` boolean check. The reason returned drives both the log entry and the F2 badge tooltip.

### Badge state machine (F2)

The toolbar icon badge is **per-tab** (`chrome.action.setBadgeText({ tabId })`) and is set by the background service worker. Content scripts notify the background of state changes; background also listens to `chrome.storage.onChanged` to react when settings change.

Badge visual states:

| State | Badge text | Badge color | Tooltip suffix |
|---|---|---|---|
| Off-domain (not Moodle/Panopto) | `''` | — | (default title) |
| Active, will auto-click | `''` | — | `Will auto-click on this page.` |
| Site disabled in settings | `OFF` | gray `#71717A` | `Disabled in settings for this site.` |
| Timed pause (F3) | `II` | orange `#F59E0B` | `Paused for N more minutes.` |
| Logout pause (session) | `II` | orange | `Paused after sign-out (all tabs).` |
| Logout pause (tab) | `II` | orange | `Paused after sign-out (this tab).` |
| Manual pause (D2) | `II` | orange | `Paused on this tab.` |

Notes:
- "II" is two ASCII capital-i's used as a pause glyph (renders as `‖` visually inside the small badge).
- Badge color via `chrome.action.setBadgeBackgroundColor({ tabId, color })`.
- The "Will auto-click after Xs" addendum is appended to the tooltip when `click_delay_ms > 0`.
- Badge updates fire on: page load (content script → background message), settings change (storage.onChanged), pause toggle (popup → background), tab switch (`chrome.tabs.onActivated` is *not* needed — badges are sticky per-tab).
- We do NOT use `chrome.alarms` to clear the timed pause. Instead, the badge accuracy is best-effort: when the popup opens or the page navigates, expiry is checked and the badge updates. This avoids needing the `alarms` permission.

### Component responsibilities

- **content.js**: detects login/logout/authenticated pages (unchanged); runs the new 7-step suppression evaluation; honors `click_delay_ms`; reads/writes the new `sessionStorage` flag for D2; messages background with state info for badge + log writes.
- **background.js**: owns badge state for all tabs; handles `login_clicked` → notification + log write + counter increment + badge update; handles new message types `state_update` (from content script) and `manual_pause_toggle` (from popup); listens to `chrome.storage.onChanged` to update badges when settings or `pause_until` change.
- **settings.html / settings.js (popup)**: gains 5 new UI elements (delay dropdown, pause-this-tab button, quick-pause control, clicks-saved footer, activity-log section); auto-follows `prefers-color-scheme: dark`; queries active tab for D2 button state on render.

## Per-Feature Specs

### D2 — Manual pause this tab

**UI:** A new group "This tab" between "Sites" and "Behaviour" in the popup. Contains one button:
- When tab is unpaused: button text "Pause on this tab", primary style.
- When tab is paused (any reason: D2 manual, logout-tab, logout-session, or timed): button text "Resume on this tab", outline style. Below the button, a one-line caption explains *why* it's paused (mirrors badge tooltip).
- When the active tab is not a Moodle/Panopto page: button is disabled with caption "Open Moodle or Panopto to use this."

**Behavior:**
- Click "Pause on this tab" → popup sends `{ type: 'manual_pause_set', value: true }` to active tab's content script.
- Content script sets `sessionStorage[skipcam_manual_pause] = '1'`.
- Content script messages background to refresh badge for this tab.
- Click "Resume on this tab":
  - If only D2 manual pause is active → popup sends `manual_pause_set: false`, content script clears the key, badge clears.
  - If multiple suppression sources are active → "Resume" only clears D2 manual pause; other sources (logout, timed) still keep the tab paused. Caption updates to reflect remaining reason. To clear logout suppression, user must click the login button manually (existing v1.0 behavior). To clear timed pause, see F3.

**Edge cases:**
- Tab refresh: `sessionStorage` survives refresh, so the manual pause persists. Good (matches user expectation).
- Tab close + reopen: `sessionStorage` is gone, manual pause cleared. Acceptable (it's "this tab" not "this URL").
- User opens popup before content script has loaded: button shows "Loading..." for ~200ms, then resolves.

### D3 — Click delay

**UI:** New row in "Behaviour" group. Label "Delay before click", dropdown with options:
- "Instant" (value `0`, default)
- "0.5 seconds" (value `500`)
- "1 second" (value `1000`)
- "3 seconds" (value `3000`)

**Behavior:**
- Stored as `click_delay_ms` in `storage.sync`.
- In `content.js`, replace `tryClick(site)` direct invocation with `setTimeout(() => tryClick(site), click_delay_ms)` after the suppression check passes.
- During the delay, the page shows normally — user can navigate away to abort.
- If the user clicks the login button manually during the delay, our pending click still fires harmlessly (no-op since page has navigated). No coordination needed.
- Delay applies to both the immediate click path and the MutationObserver-found-button path.

**Why no per-site setting:** the use cases are global (slow networks, debug visibility). Adding per-site dials doubles storage keys for a setting most users will set once.

### F1 — Dark mode popup

**Behavior:** Auto-follow `prefers-color-scheme`. No manual toggle. Implementation is pure CSS — add a `@media (prefers-color-scheme: dark)` block that overrides the existing `:root` CSS custom properties.

**Dark palette (mirroring existing token names):**
```css
@media (prefers-color-scheme: dark) {
  :root {
    --bg: #18181B;
    --bg-footer: #09090B;
    --text: #FAFAFA;
    --text-secondary: #D4D4D8;
    --text-muted: #A1A1AA;
    --text-footer: #D4D4D8;
    --border: #27272A;
    --border-subtle: #1F1F23;
    --accent: #10B981;          /* green stays — high contrast on dark */
    --accent-soft: rgba(16, 185, 129, 0.25);
  }
  select { background-color: #27272A; }
  /* Toggle slider unchecked state needs darker bg in dark mode */
  .toggle .slider { background: #3F3F46; }
}
```

**Contrast constraint:** All text token values in the dark palette must achieve WCAG AA contrast (4.5:1 for normal-weight text, 3:1 for large/bold). Specifically `--text-muted` is consumed by `.group-label` at 10.5px, which does not qualify as "large text" at any weight.

**Non-goals:** No transitions on theme switch (Chrome reloads the popup on each open anyway).

### F2 — Toolbar icon badge for state

(See "Badge state machine" in Architecture Overview.)

**Implementation summary:**
- Background service worker is the single source of truth for what the badge shows on each tab.
- Three triggers refresh a tab's badge:
  1. Content script sends `{ type: 'state_update', state: <state-name>, reason?: <string> }` on page load and whenever its local pause state changes.
  2. Background's `storage.onChanged` handler re-evaluates *all currently-tracked tabs* when relevant keys change (`<site>_status`, `pause_until`, `suppress_auto_login`, `logout_scope`).
  3. Popup sends `{ type: 'refresh_badge', tabId }` after toggling pauses.
- Background maintains an in-memory `Map<tabId, { state, reason, expiresAt? }>` for fast re-evaluation. Cleared on `chrome.tabs.onRemoved`.
- Tooltip set via `chrome.action.setTitle({ tabId, title })`.

**Why no `alarms` for timed-pause auto-clear:** When the badge would show a stale "II" after `pause_until` elapses on an idle tab, that's purely cosmetic. The next interaction (page navigation, popup open, settings change) re-evaluates and clears it. Not worth a permission.

### F3 — Quick timed pause from popup

**UI:** New group "Quick actions" below "Behaviour". Layout:
- One row containing: a `<select>` with options "15 minutes" / "1 hour" / "4 hours" (default "1 hour"), and a button "Pause everywhere".
- When a timed pause is active, this row is replaced with a status block: "Paused for N more minutes" + a "Cancel" button.

**Behavior:**
- Click "Pause everywhere" → popup writes `pause_until = Date.now() + selected_duration_ms` to `storage.session`. Background's `storage.onChanged` fires → re-evaluates all tracked tabs → updates badges to "II" with appropriate tooltip.
- Auto-click suppression checks `pause_until > Date.now()` in priority step 2 of evaluation.
- Click "Cancel" → popup removes `pause_until` from `storage.session` → badges refresh.
- Pause persists across tab navigations within the browser session. Cleared automatically when browser closes (`storage.session` semantics) or when expiry passes (lazy-cleared on next read).

**Why "until I unpause" is excluded:** Forgettable. Indefinite pause is what site toggles in settings already provide.

### F4 — Clicks-saved counter

**UI:** A thin row sits directly above the existing BMC/GitHub footer, separated by the same `--border` divider. Layout:
- Display: "247 clicks saved" (small text, `--text-muted` color, centered).
- If count is 0: hide the row entirely (no zero-state).
- Tooltip on hover: "Total auto-clicks since installing".

Reasoning: the existing footer is already two-column (BMC + GitHub) and crowded; adding the counter as its own row keeps spacing clean.

**Storage:** `chrome.storage.local.clicks_saved`, integer, increments by 1 in background's `handleLoginClicked` after a successful click. Never decremented, no reset UI in v1.1.

**Why local not sync:** Prevents quota churn (sync has 8KB-per-key + 1MB total + ~120 ops/min limits — fine for one integer, but we're already putting `activity_log` in local, so colocation simplifies the mental model). Trade-off: counter doesn't follow you across devices. Acceptable for a vanity metric.

### F5 — Recent-activity log

**UI:** New collapsible group "Recent activity" at the bottom of the popup, above the footer. Collapsed by default.
- Header row: "Recent activity" + chevron icon. Click toggles expand/collapse.
- Expanded: rendered list of up-to-50 entries, newest first. Each entry: `<relative-time> · <site> · <action-label>`.
- Action labels: "Auto-clicked", "Suppressed (reason)", "Failed (reason)".
- Empty state: "No activity yet."
- Below the list, small "Clear log" link (left-aligned, muted color).

**Storage:** `chrome.storage.local.activity_log`, array. Background's log helper:
```js
async function appendLogEntry(entry) {
  const { activity_log = [] } = await chrome.storage.local.get('activity_log');
  activity_log.unshift(entry);
  if (activity_log.length > 50) activity_log.length = 50;
  await chrome.storage.local.set({ activity_log });
}
```

Race conditions on concurrent writes (two tabs auto-clicking simultaneously) are accepted as best-effort — we may lose one entry occasionally. Not worth a mutex.

**Privacy:** Log stays local, never synced, never transmitted. URLs are not stored — only the site name (`'Moodle'` or `'Panopto'`) and action label.

## Inter-feature interactions

| If active | Click happens? | Badge | F4 increments? | F5 logs as |
|---|---|---|---|---|
| Site disabled | No | OFF | No | `suppressed/site-disabled` |
| Timed pause (F3) | No | II | No | `suppressed/timed-pause` |
| Logout pause | No | II | No | `suppressed/logout-*` |
| Manual pause (D2) | No | II | No | `suppressed/manual-tab` |
| All clear | Yes (after delay) | empty | Yes | `clicked` |
| Click attempt fails | No | empty | No | `failed/<reason>` |

D2 manual-pause and F3 timed-pause stack: clearing one doesn't clear the other. The popup's UI shows whichever pause(s) apply to the current tab.

## Testing approach

This is a Chrome extension with no test framework currently installed. v1.1 will include a manual test plan in `docs/superpowers/specs/2026-04-28-v1.1-test-plan.md` (created during implementation). Each feature gets a checklist of scenarios to verify in a clean Chrome profile loaded with the unpacked extension. No automated tests in v1.1; that's a separate initiative if pursued.

## Migration / backwards compatibility

- v1.0 → v1.1 upgrade path: existing settings (`moodle_status`, `panopto_status`, `notification_status`, `logout_scope`) are read unchanged. New keys default to v1.0-equivalent behavior on first read:
  - `click_delay_ms` defaults to `0` → instant (matches v1.0).
  - `clicks_saved` defaults to `0` (lifetime starts at v1.1 install/upgrade — we don't backfill).
  - `activity_log` defaults to `[]`.
  - `pause_until` absent → no timed pause.
  - `skipcam_manual_pause` absent → no manual pause.
- No `manifest.json` permission changes → no upgrade prompt for users.
- `manifest.json` version bumps to `1.1`.
